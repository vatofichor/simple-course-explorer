# Simple Course Explorer & Content Creator Studio

Welcome to **Simple Course Explorer**, a retro-premium, lightweight, offline-capable learning management system and content creator studio. 

Course Explorer is designed to run in two primary modes:
1. **Desktop Native Mode:** Optimized for parents, educators, and individual users running locally on a personal computer (no technical expertise or server hosting required).
2. **Staging / Web Server Mode:** Designed for developers and content organizations hosting a multi-user staging site or public web server.

---

## Quick Start (Desktop Native Mode)

If you are a parent, teacher, or student running Course Explorer on a Windows or Linux/macOS desktop, follow these simple steps:

### 1. Installation
1. Locate the installation script in the project root:
   - **Windows:** Double-click `install.bat`.
   - **Linux/macOS:** Open a terminal and run `./install.sh`.
2. Follow the on-screen prompts. The installer will:
   - Setup administrative configurations.
   - Prompt you to enter a new admin password (this protects your local Course Explorer from unauthorized changes).
   - Generate your local configuration.

### 2. Starting the Course Explorer
Once installed, starting the application is simple:
- **Windows:** Double-click `run-server.bat` in the root folder.
- **Linux/macOS:** Run `./run-server.sh`.
- Your default web browser will automatically open to: `http://localhost:8000/`

To close the server, simply close the command-prompt window or terminal.

### 3. Updating Your Admin Password
If you ever need to change or reset your administrator password:
- **Windows:** Go to the folder `dev/admin_scripts/` and double-click `update_password.bat`.
- **Linux/macOS:** Navigate to `dev/admin_scripts/` and run `./update_password.sh` in the terminal.

## Advanced Operations (Web Server / Hosting Mode)

For web administrators, systems engineers, and developers deploying Course Explorer on remote host environments:

### 1. Requirements & Dependencies
- **Core Engine:** PHP (compatible with PHP 7.4 through PHP 8.x). No database required.
- **Client Render Engine:** JavaScript (ES3+ compatible).
- **Environment:** Apache Web Server (recommended, support file `.htaccess` included).

### 2. Production Deployment (Build Filter)
When deploying to a public production web server, ensure you block or exclude administrative tooling from public access:
- **Keep in Production:** `/public/`, `/admin/`, `config.php` (holds hashed passwords, blocked by `.htaccess`).
- **Exclude from Production:** `/dev/` (contains specs, uncompiled source files, CLI tools).
- **Security Check:** Ensure the `.htaccess` rules are actively enforced by Apache to prevent direct hits on config or source trees.

### 3. Rebuilding Markdown Content
The public website reads compiled HTML structures. To rebuild files on a server after modifying markdown source articles inside `/content_source/`:
1. Log in to the host machine via SSH.
2. Navigate to the project root and execute the builder script:
   ```bash
   php dev/admin_scripts/convert.php
   ```
This updates the public lesson structures and updates `/public/content/content_manifest.json`.

## Project Repository Boundaries

Here is how the system boundaries are structured:
- **`content_source/`**: Contains raw markdown files organized into sections (edited via the page editor).
- **`public/content/`**: Contains compiled HTML outputs and the index manifest generated by the compiler.
- **`admin/`**: Houses PHP scripts for dashboard navigation, lesson editing, and authentication logic.
- **`dev/admin_scripts/`**: Holds CLI-only configuration scripts (such as password updaters, builders, and backups).
- **`dev/specs/`**: Developer specifications including setup documentation and security subsystem specifications.
